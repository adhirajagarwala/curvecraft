# v0.3.0-mosfet-id-vds-rdson Release Notes

Tag: `v0.3.0-mosfet-id-vds-rdson`  
Commit: `6e824243dd994b873cc6f1f856f7fb834a5fedf5`

This release completes CurveCraft M3: educational nMOS MOSFET Id-Vds
output-curve fitting, low-Vds `Rds_on` extraction, and ngspice implementation
validation. It builds on the completed M1 diode and M2 MOSFET Id-Vgs workflows.

## What Changed

- Added the MOSFET Id-Vds output-curve workflow for bundled CSV data.
- Added simplified Level-1 / square-law Id-Vds fitting for nMOS output curves.
- Added low-Vds `Rds_on` extraction by Vgs curve.
- Added M3 plots, Markdown report generation, generated ngspice netlists,
  ngspice output artifacts, and Python-vs-ngspice comparison artifacts.
- Preserved M1 diode I-V and M2 MOSFET Id-Vgs workflows.

## Validation Meaning

CurveCraft validation separates:

```text
source curve data
-> fitted Python compact model
-> generated ngspice model/netlist
-> Python-vs-ngspice comparison
```

The source curve data is the input being fitted. The Python result is
CurveCraft's fitted compact-model output. The generated ngspice files are a
separate simulator representation of the same simplified model. The comparison
checks implementation consistency between Python and ngspice outputs; it does
not prove real-world device truth or production model validity.

## Limitations

- CurveCraft uses educational compact models, not production-grade model
  extraction.
- M2 and M3 use simplified Level-1 / square-law MOSFET modeling, not BSIM.
- MOSFET support is nMOS only in the current scope.
- Threshold voltage and `Rds_on` are extraction-method-dependent results.
- ngspice validation checks implementation consistency, not physical truth.
- The project does not include pMOS, BSIM, capacitance, gate-charge,
  switching-loss, thermal modeling, or datasheet image digitization support.
- Fitted parameters should not be used for engineering decisions without data
  provenance, measurement conditions, model validity bounds, and independent
  validation.
